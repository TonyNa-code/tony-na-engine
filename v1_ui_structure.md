# Tony Na Engine V1 界面结构设计

## 1. 这份文档是干什么的

这份文档专门回答一个问题：

> 第一版引擎，界面到底长什么样，用户具体点哪里来做游戏？

目标依旧不变：

- 不让用户写代码
- 尽量减少复杂设置
- 让“上传素材 + 输入台词 + 点击按钮”就能做出可玩的 galgame

## 2. V1 总界面结构

第一版编辑器采用固定的 5 个主页面。

1. 首页
2. 剧情编辑器
3. 资源库
4. 角色管理
5. 预览与导出

顶部导航建议固定显示这 5 个入口，避免用户迷路。

推荐导航文案：

- 首页
- 写剧情
- 管素材
- 管角色
- 预览导出

不要写成：

- Dashboard
- Asset Manager
- Scene Graph

因为这些词对新手不友好。

## 3. 首页设计

首页的任务只有三个：

- 新建项目
- 打开已有项目
- 导出当前项目

### 首页布局

- 左上：项目名称、封面图、最后编辑时间
- 左下：快速开始卡片
- 右侧：最近项目列表
- 底部：导出状态、项目检查结果

### 首页主要按钮

- 新建游戏
- 打开游戏
- 继续上次编辑
- 进入剧情编辑
- 一键试玩
- 导出游戏

### 新建项目弹窗

字段尽量少：

- 游戏名称
- 游戏分辨率
- 文本语言
- 是否使用模板

模板建议先给 4 个：

- 校园恋爱
- 奇幻冒险
- 悬疑推理
- 空白项目

## 4. 剧情编辑器设计

这是整套引擎最核心的页面。

### 剧情编辑器布局

- 左侧：章节和场景树
- 中间：剧情卡片时间线
- 右侧：当前卡片设置区
- 顶部：常用插入按钮
- 底部：当前场景即时预览条

### 左侧区域：章节和场景树

功能：

- 新建章节
- 新建场景
- 复制场景
- 删除场景
- 拖动排序

推荐按钮文案：

- 新建章节
- 新建场景
- 复制这个场景
- 删除这个场景

### 中间区域：剧情卡片时间线

这一栏用“卡片”而不是“代码行”来表达剧情。

V1 卡片类型：

- 台词
- 旁白
- 选项
- 跳转
- 切换背景
- 显示角色
- 隐藏角色
- 播放音乐
- 停止音乐
- 播放音效
- 设置变量
- 修改变量
- 条件判断

每张卡片都要能看见最重要的信息。

比如：

台词卡片应直接显示：

- 角色名
- 台词前几句
- 绑定语音

选项卡片应直接显示：

- 选项 1 文案
- 选项 2 文案
- 每个选项会跳到哪里

### 顶部插入工具条

这是给新手最常用的入口。

推荐按钮顺序：

- 加台词
- 加旁白
- 加选项
- 切背景
- 显角色
- 播音乐
- 更多

把复杂功能收进“更多”，不要一股脑全摆出来。

### 右侧区域：当前卡片设置区

当用户点中不同卡片，右侧显示不同表单。

例如点中“台词卡片”时，右侧显示：

- 说话角色
- 角色表情
- 台词文本
- 绑定语音
- 是否自动下一句
- 打字速度

例如点中“选项卡片”时，右侧显示：

- 选项数量
- 每个选项文案
- 每个选项跳转到的场景
- 选择后变量变化

### 底部即时预览条

这是一个非常重要的减压设计。

用户改完一句话，就能马上看到：

- 角色名框效果
- 对话框效果
- 当前背景
- 当前立绘位置

让新手不需要频繁切页。

## 5. 资源库设计

资源库的目标不是“像文件系统”，而是“像素材超市”。

### 分类

- 背景
- 立绘
- 表情
- CG
- 音乐
- 音效
- 语音
- 视频
- 界面素材

### 资源库布局

- 左侧：素材分类
- 中间：缩略图网格
- 右侧：素材信息和使用位置

### 主要操作

- 导入文件
- 批量导入
- 预览
- 重命名
- 替换素材
- 查看这个素材被哪些场景使用

### 新手友好规则

- 图片尽量显示缩略图，不要只显示文件名
- 音频要支持一键试听
- 缺失素材要有红色提醒
- 替换素材后，场景应自动继承，不要要求用户重新绑定

## 6. 角色管理页面设计

角色页的目标是让用户先把人物准备好，后面写剧情时就能直接选。

### 每个角色的字段

- 角色名称
- 显示颜色
- 默认站位
- 角色简介
- 默认立绘
- 表情列表
- 常用语音映射

### 页面布局

- 左侧：角色列表
- 中间：角色信息
- 右侧：表情和立绘缩略图区

### 主要按钮

- 新建角色
- 复制角色
- 添加表情
- 设置默认立绘
- 删除角色

## 7. 预览与导出页面设计

这个页面让用户感受到“我真的已经在做游戏了”。

### 页面布局

- 左侧：试玩窗口
- 右侧：导出设置和检查结果

### 试玩相关按钮

- 从开头试玩
- 从当前场景试玩
- 从当前卡片试玩
- 模拟选择分支
- 清空临时存档

### 导出相关按钮

- 检查项目问题
- 导出 Windows 版
- 导出网页试玩版
- 打开导出文件夹

### 导出前检查项

- 有没有缺失图片
- 有没有缺失音频
- 有没有跳转错误
- 有没有空白选项
- 有没有未命名场景

检查结果不要只写“失败”，要写清楚：

- 错在哪
- 在哪个场景
- 点哪里去修

## 8. V1 最推荐的新手工作流

如果用户第一次打开软件，最理想的流程应该是：

1. 点击“新建游戏”
2. 选“校园恋爱模板”
3. 添加 2 个角色
4. 导入 1 张背景和 2 张立绘
5. 在剧情编辑器里输入第一段对话
6. 插入一个双选项
7. 点击“从当前场景试玩”
8. 点击“导出 Windows 版”

如果这个流程顺畅，说明引擎方向是对的。

## 9. 关键按钮的人话文案建议

下面这些文案会直接影响新手体验。

推荐：

- 这个选项会去哪里？
- 这句台词是谁说的？
- 请选择要显示的背景
- 请选择这句台词的语音
- 这个角色站在左边、中间还是右边？
- 这个场景还没有名字，先给它起个名吧

不推荐：

- Bind target
- Reference asset
- Resolve dependency
- Character anchor
- Validate graph

## 11. 现在最值得马上做的开发顺序

如果我们下一步开始真正制作原型，最建议的顺序是：

1. 先做首页和主导航
2. 再做剧情编辑器框架
3. 先支持“台词卡片”和“选项卡片”
4. 再补资源库
5. 再补预览功能
6. 最后做导出

原因很简单：

剧情编辑器是整套引擎的心脏，只要它跑通，整套工具就开始有生命了。
